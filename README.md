# CampusQuickServe_ICE

## Problem Description
This program simulates a simple ordering system for a campus food vendor.  
The system collects customer and order information, calculates totals including VAT and possible discounts, and displays a formatted receipt.

## Program Structure

### Classes
Order
- Constructor
- calculateSubtotal()
- calculateDiscount()
- calculateVAT()
- calculateTotal()

QuickServeApp
- main()

## Development Approach
The program was developed using object-oriented programming principles.  
An Order class was created to handle order data and calculations, while the main application class handled user interaction through JOptionPane.

## OOP Concepts Used

Class vs Object
- Order is the class
- order is the object created in the main program.

Encapsulation
- Attributes are private with getters.

Constants
- VAT_RATE constant used for VAT calculations.

Methods
- Methods used for subtotal, discount, VAT and total calculations.
